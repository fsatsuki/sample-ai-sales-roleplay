# Integration Test Instructions - Nova 2 Sonic移行

## 目的
Nova 2 Sonic BidiAgentとフロントエンド間のリアルタイム音声対話フローを検証する。

## 前提条件
- CDKデプロイ完了（`npm run deploy:dev`）
- フロントエンド環境変数設定完了（VITE_NOVA_SONIC_AGENT_ENDPOINT, VITE_NOVA_SONIC_AGENT_REGION）
- Cognito認証ユーザーでログイン可能

## テストシナリオ

### シナリオ1: 基本的な音声対話フロー
1. シナリオ選択画面でシナリオを選択
2. 会話画面を開く
3. マイクボタンをクリックして音声入力を開始
4. 日本語で発話する
5. **期待結果**:
   - ASR転写テキストがチャットUIにリアルタイム表示される
   - NPC応答テキストがチャットUIに表示される
   - Polly音声合成でNPC応答が読み上げられる
   - リアルタイムスコアリング（怒り・信頼・進捗メーター）が更新される

### シナリオ2: テキスト入力（Cross-modal Input）
1. 会話画面でテキスト入力フィールドにメッセージを入力
2. 送信ボタンをクリック
3. **期待結果**:
   - テキストがNova 2 Sonicに送信される
   - NPC応答テキストが返される
   - Polly音声合成でNPC応答が読み上げられる

### シナリオ3: endpointingSensitivity設定
1. 設定パネルでターン検出感度をHIGHに変更
2. 音声対話を実施
3. **期待結果**: 短い間（ポーズ）でもターン切り替えが発生する
4. 設定をLOWに変更して再度音声対話
5. **期待結果**: 長めの間でもターン切り替えが発生しにくい

### シナリオ4: セッション切断・再接続
1. 音声対話中にネットワークを一時切断（またはブラウザタブを非アクティブに）
2. ネットワーク復帰後
3. **期待結果**:
   - エラーメッセージが表示される
   - セッション再開ボタンが表示される
   - 再開ボタンクリックで新しいセッションが開始される

### シナリオ5: Session Continuation（6分超セッション）
1. 6分以上の長時間セッションを実施
2. **期待結果**:
   - 6分経過時にバックグラウンドで次セッションが準備される
   - ユーザーに中断なくセッションが継続される
   - 会話履歴が維持される

### シナリオ6: 廃止機能の非存在確認
1. 会話画面を開く
2. **期待結果**:
   - silenceThreshold設定UIが存在しないこと
   - TranscribeService関連のWebSocket接続が発生しないこと
   - API Gateway WebSocketエンドポイントへの接続が発生しないこと

## API連携テスト

### AgentCore Runtime接続確認
- フロントエンドからSigV4署名付きWebSocket接続が確立されること
- Cognito一時認証情報が正しく取得されること
- WebSocketメッセージの送受信が正常に動作すること

### AgentCore Memory連携確認
- 会話履歴がAgentCore Memoryに保存されること
- セッション終了後に履歴が取得可能であること

### スコアリングAPI連携確認
- ASR転写テキスト受信後にスコアリングAPIが呼び出されること
- スコアリング結果がUIに反映されること

## ブラウザ互換性テスト
- Chrome（最新版）: 必須
- Firefox（最新版）: 推奨
- Safari（最新版）: 推奨
- Edge（最新版）: 推奨

確認事項:
- AudioWorklet（NovaSonicAudioProcessor）が正常に動作すること
- WebSocket接続が正常に確立されること
- マイク入力が正常にキャプチャされること
